## 1. 项目背景

### 1.1 背景说明

杭州金源电器有限公司主营互感器类产品，现有《互感器产品选型手册》（PDF，44 页），涵盖企业介绍、资质证书、18 个产品系列及完整技术参数。

当前缺少统一的 **线上展示入口**。销售经理在外出拜访客户时，需要一种正式、专业、便于在手机/电脑上快速展示的公司门面，以建立客户（尤其是企业决策层）对公司的初步信任。

### 1.2 项目目标

| 目标类型 | 描述 | 衡量指标 |
|----------|------|----------|
| **品牌展示** | 向 B 端客户展示公司正规性与制造实力 | 页面完整度、视觉一致性 |
| **信任建立** | 3 分钟内让决策人形成「靠谱厂家」印象 | 资质页、关于页内容覆盖率 |
| **销售赋能** | 销售可微信转发链接，现场打开展示 | 移动端适配、首屏加载 ≤ 3s |
| **资料触达** | 引导客户下载完整 PDF 样本册 | PDF 下载按钮点击率 |
| **联系转化** | 客户能便捷找到电话、地址、二维码 | 联系页信息完整、一键拨号可用 |

### 1.3 非目标（本期不做）

- 搜索引擎大规模引流（SEO 长尾布局）
- 在线商城 / 在线下单
- 复杂产品参数筛选与型号搜索引擎
- 18 个产品型号各自独立详情页（参数留在 PDF）
- 用户注册 / 会员系统
- 英文版（二期可选）
- 新闻资讯 / 博客 CMS
- CRM / ERP 系统对接

## 2. 用户与场景

### 2.1 目标用户

| 用户角色 | 典型身份 | 核心诉求 | 使用设备 |
|----------|----------|----------|----------|
| **Primary — 企业决策人** | 公司总经理、采购总监 | 快速判断厂家是否正规、有实力、可合作 | 手机（微信链接）为主 |
| **Secondary — 销售经理** | 本公司销售人员 | 拜访时展示、微信转发、下载 PDF 发给客户 | 手机 + 笔记本 |
| **Tertiary — 技术人员** | 客户方电气工程师 | 获取完整参数（通过 PDF，非网站） | 手机 / PC |

### 2.2 核心使用场景

**场景 A：拜访展示**
> 销售经理携带笔记本或平板，打开官网首页向客户老总介绍公司，随后展示资质认证页。

**场景 B：微信转发**
> 销售将官网链接发送至客户微信，决策人在手机上浏览首页 + 产品大类 + 资质页。

**场景 C：资料索取**
> 客户有意向后，销售引导其点击「下载样本册 PDF」，或将 PDF 直接发送给客户的技术人员。

**场景 D：后续联系**
> 客户通过联系页获取电话、地址，或直接点击「一键拨号」。

## 5. 非功能需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 首屏加载时间 | ≤ 3 秒（4G 网络） |
| 页面总大小 | 首页 ≤ 2MB（不含 PDF） |
| PDF 文件大小 | 当前约 8.7MB，可接受；如有优化需求可压缩 |

### 5.2 兼容性

| 平台 | 最低版本 |
|------|----------|
| Chrome | 最新两个大版本 |
| Safari（iOS） | iOS 14+ |
| 微信内置浏览器 | 必须兼容 |
| Edge / Firefox | 最新两个大版本 |

### 5.3 安全

- 全站 HTTPS
- 留言表单需防刷（简单验证码或频率限制）
- 不收集不必要的用户隐私数据

### 5.4 可维护性

- 产品系列内容可通过配置文件或 CMS 修改（建议 JSON/Markdown 管理，无需改代码）
- 证书图片、产品图可替换
- PDF 文件可独立更新替换
