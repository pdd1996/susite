# ADR-0001：文档权威来源与同步规则

| 项目 | 内容 |
| --- | --- |
| 状态 | 已采纳 |
| 日期 | 2026-07-15 |
| 决策范围 | 展站平台、SiteConfig 契约与金源样板站文档 |

## 背景

展站同时包含平台规划、机器可读配置契约与客户样板文档。若没有明确的权威来源和同步规则，页面范围、素材状态、字段约束和技术实现容易产生多份互相矛盾的说明。

## 决策

| 主题 | 权威来源 | 同步要求 |
| --- | --- | --- |
| 平台范围、功能、API、数据模型、验收标准 | [展站 PRD](../展站-产品需求文档(PRD).md) | 修改范围或验收标准时同步项目计划 |
| 阶段、进展、下一步行动 | [展站计划](../展站计划.md) | 不得用计划替代 PRD 的功能定义 |
| `SiteConfig` 字段、类型、数量限制 | [SiteConfig v1 Schema](../schemas/site-config-v1.schema.json) | 破坏性变更升级 `schemaVersion`，并同步 PRD 与迁移文档 |
| Demo 与 V1 的字段和运行模型差异 | [Demo 迁移到 V1](../migration-demo-to-v1.md) | Schema 或 Demo 结构变化时检查字段映射 |
| 金源页面范围、素材状态、验收 | [金源官网 PRD](../jinyuan/金源电器官网-产品需求文档(PRD).md) | 同步金源 README 的当前范围和待补齐项 |
| 金源色彩、组件、布局与交互 | [金源设计规范](../jinyuan/金源电器官网-设计规范.md) | 视觉调整时同步 PRD 中的品牌规范 |
| 金源首页定稿文本 | [金源首页文案](../jinyuan/金源电器官网-首页文案.md) | 内容结构变化时同步 PRD 与设计规范 |

## 冲突处理

1. 先按上表确定冲突主题的权威来源。
2. 更新权威文档，再同步引用该事实的其他文档。
3. 无法立即确认的客户信息标为“待确认”或“占位”，不得标为已交付。
4. 历史讨论记录只保留背景价值，必须声明不作为范围和验收依据。

## 后果

- 根目录 `README.md` 只作为仓库入口和导航，不重复承载详细规范。
- `docs/jinyuan/README.md` 作为客户子域索引保留。
- 当前不新增 `docs/README.md`；当 `docs/` 出现多个一级领域且文档量明显增长时再评估。
- 平台代码建立后，Schema 的运行时实现迁入 `packages/site-config`，文档版 Schema 与代码版必须保持一致。
