## 背景

Phase 1 基线已能构建和运行，但审查发现配置契约漂移、写接口身份可伪造、MySQL 错误映射不完整、模板未消费 Revision，以及后台缺少版本历史能力。

## 目标

- JSON Schema 与 Zod 对未知字段、唯一数组和业务引用保持一致；
- 本地开发身份不可由客户端任意伪造，生产模式必须接入可信身份；
- MySQL 唯一冲突、非法 JSON 和数据库关系得到明确处理；
- 模板可直接读取指定 Revision 配置；
- 后台可选择站点、编辑核心配置并查看 Revision 历史。

## 非目标

- 接入正式 IDaaS；
- 实现 OSS Asset、异步构建和 HTTPS 部署；
- 完整复刻金源视觉稿。

## 验收标准

- [x] 契约一致性测试覆盖未知字段、唯一数组和引用关系；
- [x] 写接口不信任客户端 `x-actor-id`，非法 JSON 返回 400；
- [x] MySQL 重复站点返回 409，表关系有外键；
- [x] 模板支持从 JSON Revision fixture 构建；
- [x] 后台可查看 Revision 历史并处理并发冲突；
- [x] `pnpm check` 与 `pnpm build` 通过。

## 验证结果

- `pnpm check`：通过，配置 4 项、API 3 项、模板 2 项、后台 1 项测试全部通过；
- `pnpm build`：通过；
- 已知环境提示：当前 Node.js 20.18.0 低于仓库声明的 20.19.0，需升级开发环境。
