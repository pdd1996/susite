## 背景

归档 Spec 审查发现：权威 JSON Schema 与 Zod 运行时校验对分类唯一性、精选分类引用和空白文本的接受范围不一致；测试未能捕获这些差异；长期文档的 Phase 1 状态和归档路径也存在过期表述。

## 目标

- 让 JSON Schema 明确表达所有可表达的 SiteConfig v1 约束，并让 Zod 的输入接受范围与其一致；
- 用同一组正反例验证 JSON Schema 与 Zod，防止契约再次漂移；
- 更正长期文档中的 Phase 1 状态与归档路径，并保留 Phase 2/3 遗留项的事实；
- 记录本次修复的验收和长期文档回写。

## 非目标

- 改变 SiteConfig v1 的字段、Schema 版本或对外 API；
- 引入正式身份服务、MySQL 集成测试环境或端到端浏览器测试；
- 重写既有归档包的历史记录。

## 影响范围

- 产品：Phase 1 状态表述与已实现范围对齐。
- 配置契约：补充 JSON Schema 中的跨字段约束，并统一文本空白处理语义。
- 数据库：无变化。
- API：无接口变化；API 继续使用统一的 Zod 契约。
- 模板 / 后台：无行为变化。
- 文档：更新 README、项目计划、PRD 及计划内归档路径。

## 验收标准

- [x] Schema 与 Zod 都拒绝空白必填文本、重复分类 ID/slug 及不存在的精选分类引用。
- [x] Schema 与 Zod 都接受现有金源 Revision fixture，且契约测试同时断言两者。
- [x] README、PRD 与项目计划均反映 Phase 1 已完成以及已登记的后续遗留项。
- [x] 项目计划使用 ADR-0002 指定的归档路径。
- [x] `pnpm check` 与 `pnpm build` 通过。

## 验证结果

- 测试：`pnpm check` 通过；配置契约 6 项、API 3 项、模板 2 项、后台 1 项测试全部通过。
- 构建：`pnpm build` 通过。
- 人工验收：README、PRD、项目计划与归档包记录均已核对。
